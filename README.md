# Moderation System

Проект реализует систему модерации обращений клиентов на основе событийной архитектуры.
Обращения поступают через Kafka, проходят проверку бизнес-правил и либо допускаются
к дальнейшей обработке, либо отклоняются.

## Архитектура

Система состоит из двух сервисов:

- **moderation-service** — основной сервис модерации.
  Принимает события из Kafka, обогащает их данными и принимает решение
  о дальнейшем прохождении обращения.

- **enrichment-service** — вспомогательный сервис.
  Предоставляет дополнительную информацию по обращению через REST API,
  используя Redis как кэш.
## Взаимодействие сервисов

1. Клиентское обращение публикуется в Kafka (Topic-1).
2. moderation-service читает событие из Kafka.
3. moderation-service вызывает enrichment-service по REST API.
4. enrichment-service получает данные из Redis и возвращает ответ.
5. moderation-service применяет правила модерации.
6. Если обращение прошло проверки — оно публикуется в Kafka (Topic-2).
